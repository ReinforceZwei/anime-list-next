# client — Vite + React Frontend

## Package Identity

React 19 SPA built with Vite. Uses TanStack Router (file-based routing), TanStack Query (server state), Mantine v8 (UI), and the PocketBase JS SDK for data and real-time subscriptions. Locale is `zh-TW` (dayjs + Mantine DatesProvider).

## Setup & Run

```sh
cd client

npm install          # install deps
npm run dev          # dev server (hot reload, router codegen auto-runs)
npm run build        # tsc + vite build
npm run lint         # eslint
npm run preview      # preview production build
```

**Required env** — create `client/.env.local`:
```
VITE_PB_URL=http://127.0.0.1:8090
```

## Architecture Layers

```
src/
├── lib/pb.ts          ← PocketBase singleton + Collections enum
├── api/               ← raw pb calls, no React (pb.send for custom routes)
├── hooks/             ← TanStack Query wrappers + PB realtime subscriptions
├── components/        ← UI components, grouped by feature
├── types/anime.ts     ← all PocketBase record interfaces
├── routes/            ← TanStack Router file-based routes
└── theme.ts           ← Mantine theme overrides
```

## Patterns & Conventions

### Imports

- Always use `@/` for absolute imports (alias → `src/`):
  - DO: `import { pb, Collections } from '@/lib/pb'`
  - DON'T: `import { pb } from '../../lib/pb'`

### PocketBase Access

- The singleton is `pb` from `@/lib/pb`. Always reference collections via the `Collections` enum, never raw strings:
  ```ts
  pb.collection(Collections.Animes).getFullList(...)  // ✓
  pb.collection('animeRecords').getFullList(...)       // ✗
  ```
- Use `pb.send<T>(path, options)` for custom server routes (TMDB proxy, import/export). It auto-attaches auth tokens:
  ```ts
  // api/tmdb.ts
  pb.send<TmdbSearchItem[]>('/api/tmdb/search', { query: { query } })
  ```

### Hooks

- Each hook wraps a `useQuery` or `useMutation` from TanStack Query — see `src/hooks/useAnimeList.ts` as the canonical pattern
- Real-time sync lives in `src/hooks/useRealtimeSync.ts` — it subscribes to PocketBase events and invalidates the query cache; do not duplicate subscription logic in components
- `staleTime: Infinity, gcTime: Infinity` on the anime list query — data only refreshes via real-time events
- Query keys follow the pattern `[Collections.CollectionName, userId]`

### Components

- Feature-grouped folders under `src/components/` (e.g. `InfoCard/`, `AnimePaper/`, `modals/`)
- Each folder has an index or barrel export only when there are multiple files
- Prefer functional components with named exports:
  - DO: `export function AnimeCard(...)` — see `src/components/InfoCard/AnimeCard.tsx`
  - DON'T: default exports for components (makes refactoring harder)
- CSS Modules colocated with component: `ComponentName.module.css`

### Modals (Mantine)

- All modals are registered in `src/components/modals/index.ts` and passed to `ModalsProvider` in `__root.tsx`
- Open modals from anywhere via the Mantine `modals` context — never render them inline
- See `src/components/modals/AddAnimeModal.tsx` for the minimal pattern, `EditAnimeModal.tsx` for a form-heavy modal

### Routing (TanStack Router)

- Routes are file-based under `src/routes/`; `routeTree.gen.ts` is auto-generated — **never edit it manually**
- Auth-required pages live under `src/routes/_auth/` (prefixed layout route `_auth.tsx` handles redirect)
- To add a new page, create `src/routes/mypage.tsx` and export `Route` via `createFileRoute`; the route registers automatically on next `npm run dev`

### Types

- All PocketBase record shapes are in `src/types/anime.ts` — extend `RecordModel` from `pocketbase`
- Field names match the PocketBase schema exactly (`lowerCamelCase`)
- Status enums: `"planned" | "watching" | "completed" | "dropped" | ""`
- Download status enums: `"pending" | "downloading" | "downloaded" | ""`

## Key Files

- PocketBase client + Collections enum: `src/lib/pb.ts`
- All record types: `src/types/anime.ts`
- Root layout (providers setup): `src/routes/__root.tsx`

## Common Gotchas

- `routeTree.gen.ts` is auto-generated by the TanStack Router Vite plugin — never edit it; run `npm run dev` to regenerate
- The `@/` alias resolves to `src/` per `vite.config.ts`; TypeScript knows about it via `tsconfig.app.json`
- `pb.authStore.record?.id` is the current user's ID — always guard with `?.` as it's null when logged out
- Mantine v8 requires PostCSS setup (`postcss.config.cjs`) — do not remove it
- dayjs locale is set globally to `zh-tw` in `__root.tsx`; use `dayjs` for all date formatting
