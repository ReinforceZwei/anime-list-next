# server ‚Äî Go + PocketBase Backend

## Package Identity

PocketBase-based backend written in Go. Extends PocketBase with custom API routes (TMDb proxy, import/export) and record lifecycle hooks (auto-populate cached TMDb titles, auto-set status timestamps).

## Setup & Run

```sh
cd server

# Create .env with required values
cp .env.example .env  # or create manually

# Run in development (auto-migrate enabled when launched via go run)
go run . serve

# Run tests (none yet)
go test ./...
```

**Required env** ‚Äî `server/.env`:
```
TMDB_API_KEY=your_tmdb_v3_api_key
DISABLE_REGISTER=false   # set true to lock registration after first user
```

## Patterns & Conventions

### Module path

Go module is `github.com/ReinforceZwei/anime-list-next/server` ‚Äî all internal imports use this prefix:
```go
import "github.com/ReinforceZwei/anime-list-next/server/hooks"
```

### Custom API Routes

- Routes are registered in `routes/` as structs implementing a `Register(se *core.ServeEvent)` method
- Group custom endpoints under `/api` with `se.Router.Group("/api")`
- Use `apis.RequireAuth()` to bind group-level auth middleware ‚Äî see `routes/importexport.go`
- Return errors via `e.JSON(http.StatusXxx, ...)` or PocketBase's `apis.NewXxxError(...)` helpers
- Adding a new route:
  1. Create `routes/mything.go` with a struct + `Register` method
  2. Instantiate and call `.Register(se)` inside `app.OnServe().BindFunc(...)` in `main.go`

### PocketBase Hooks

- Hooks are in `hooks/` as structs with a `Register(app core.App)` method
- Hook into PocketBase events with `app.OnRecordCreate("collectionName").BindFunc(...)`
- Always call `e.Next()` at the end of a hook to continue the chain
- Hooks that talk to external APIs (TMDb) receive the API client via constructor injection ‚Äî see `hooks/anime_records.go`
- Errors in background-only operations (like fetching TMDb title) should be **logged, not returned** so the main record save still succeeds

### Collections & Schema

Collections (managed by PocketBase migrations):
- `users` ‚Äî built-in PocketBase users
- `animes` ‚Äî anime watchlist entries (one per user per anime/season)
- `tags` ‚Äî user-defined tags referenced by `animeRecords.tags` (relation field)
- `lastUpdates` ‚Äî tracks last-write timestamps per user per collection (for sync)
- `userPreferences` ‚Äî per-user UI preferences (labels, page title)

Field naming convention: `lowerCamelCase` ‚Äî matches the JavaScript client.

### Migrations

- Migration files in `migrations/` are auto-generated by PocketBase when running via `go run`
- Auto-migrate is enabled only when launched with `go run` (`osutils.IsProbablyGoRun()` check in `main.go`)
- To inspect current schema: read migration files in `migrations/` or use the PocketBase admin UI

### Config

- Loaded from environment via `config/config.go`
- Access config in `main.go` and pass to hooks/routes via constructors ‚Äî never read `os.Getenv` outside `config/`

### Import / Export

- `GET /api/export` ‚Äî returns all user data as `ExportData` JSON (version 1)
- `POST /api/import` ‚Äî upserts from `ExportData`; tag IDs inside the file are reference keys, not PocketBase IDs
- Upsert logic: records with `tmdbId` matched by `(tmdbMediaType, tmdbId, tmdbSeasonNumber)`; records without by `customName`
- Import runs in a single transaction (`e.App.RunInTransaction`)

## Key Files

- Entry point + wiring: `main.go`
- Config loading: `config/config.go`
- TMDb hook (cached title, status dates): `hooks/anime_records.go`
- lastUpdates hook: `hooks/last_updates.go`
- TMDb proxy routes: `routes/tmdb.go`
- Import/export routes: `routes/importexport.go`

## TMDb API Routes

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/tmdb/search` | Multi-search by query string |
| GET | `/api/tmdb/details` | Get details by `tmdbId` + `mediaType` |

Both require authentication. Implemented in `routes/tmdb.go`.

## Common Gotchas

- Auto-migrate only runs under `go run` ‚Äî running the compiled binary does NOT auto-migrate; apply migrations by running `go run . migrate up` or via `go run . serve` in dev
- The `cachedTitle`/`cachedSeasonName` hook only fires on `OnRecordCreate`, not on import (direct `txApp.Save` bypasses hooks) ‚Äî this is intentional so import preserves the exported values
- `defaultLang = "zh-TW"` in `hooks/anime_records.go` controls the language used for fetching TMDb titles; change there to affect all title fetches
- PocketBase's `record.Original()` returns the state before the current update ‚Äî use it in update hooks to compare old vs new values
- `apis.RequireAuth()` bound on a router group applies to all routes in that group
